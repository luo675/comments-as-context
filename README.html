15 Skills · Core + Extension · Composable 15 个 Skill · 核心集 + 扩展集 · 按需叠加
⚡ Pain Point ⚡ 痛点

AI modifies code with limited context — it doesn't know what code is related. Fixed one thing, broke another. Comments act as a panoramic map for the AI. AI 改代码时只看到一段，不知道关联了谁、被谁调用。 改了这里，漏了那里。加上注释，等于给 AI 装了「全景地图」。

📝 Language Decision 📝 语言决策

All comments generated by these skills MUST be in English.
Why? ① English saves ~30% tokens (directly helps the "compensate context" goal) ② Code itself is English — same language eliminates semantic translation cost ③ Universal for any project. 所有由这些 skill 生成的注释必须使用英文编写。
原因：① 节省约 30% token（直接有利于「弥补上下文」的目标）② 代码本身是英文，同语言消除语义转换损耗 ③ 普适性，可用于任何项目。
